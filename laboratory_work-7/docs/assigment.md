# BookSwap

## Платформа для обмена книгами

- Каталог: Добавление книг - название, автор, ISBN и поиск по фильтрам
- Владение: Пользователь может добавить книгу в свой "личный фонд"
- Обмен: Запрос на получение книги у другого пользователя. Фиксация истории перемещения книги
- Рейтинг: Возможность оставить отзыв и оценку книге или надежности пользователя

### Функциональные требования

1.1. Управление пользователями

- Регистрация (имя, email, пароль).
- Аутентификация (JWT/Sessions).
- Просмотр профиля пользователя с его рейтингом.

1.2. Каталог книг

- Добавление книги в общую базу (название, автор, ISBN, описание, теги).
- Поиск по каталогу (фильтрация по автору, тегам, названию).

1.3. Личный книги

- Пользователь может заявить о владении книгой из каталога.
- Управление статусом своего экземпляра: Available, Reserved, Rent, Hidden

1.4. Процесс обмена

- Создание запроса на обмен (ExchangeRequest) от одного пользователя к другому на конкретную книгу.
- Жизненный цикл запроса:
  - Pending - ожидание
  - Accepted - владелец согласен
  - Rejected - отказ владельца
  - Completed - книга передана и это зафиксировалось
  - Canceled - отказ от аренды

1.5. Система отзывов и рейтингов

- Отзывы на книгу -0-10 range + текстовый отзыв опционально
- Отзывы на пользователя после завершения обмена
- Пересчет среднего рейтинга сущностей

2 Доменная Модель

2.1 Сущность: User (Пользователь)

Отвечает за профиль, безопасность и агрегацию рейтинга.

Fields: ID, Name, Email, PasswordHash, Rating (Points, Count).

Invariants: Нельзя добавить отзыв самому себе; рейтинг пересчитывается автоматически.

2.2 Сущность: Book (Метаданные книги)

Описывает книгу как объект в мировом каталоге.

Fields: ID, Title, AuthorID, ISBN (Value Object), Description, Topics.

Invariants: ISBN должен быть валидной длины (10 или 13); описание не длиннее 1000 символов.

2.3 Сущность: OwnedBook (Экземпляр пользователя)

Связывает физическую книгу с конкретным владельцем.

Fields: ID, BookID, OwnerID, Status (Available, Reserved, Rent, Hidden).

2.4 Сущность: ExchangeRequest (Заявка на обмен)

Реализует логику взаимодействия между людьми.

Fields: ID, OwnedBookID, FromID (кто просит), ToID (у кого просят), Status, DateInfo, Note.

State Machine:

Accept(): только из Pending.

Reject(): только из Pending.

Complete(): только из Accepted.

2.5 Сущность: Review (Отзыв)

Универсальная структура для обратной связи.

Fields: ID, FromID, ToID (User или Book), Report, Mark (0-10).

3 Архитектура

- Domain: Описанные выше структуры и их методы (бизнес-правила).
- Use Cases (Usecase/Controller в твоем ТЗ): Сценарии. Например, ExchangeUseCase.RequestBook. Он берет из репозитория OwnedBook, создает ExchangeRequest и сохраняет его.
- Clients (Repository): Интерфейсы в домене, реализация в инфраструктуре. Сохраняет структуры в Postgres.
- API (Handlers): Принимает JSON через , переводит в доменные модели и отдает в Use Case.
